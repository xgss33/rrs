# rrs Coding Guidelines

## 文件格式

- 文本文件统一使用 LF；编辑时避免产生无关的整文件换行变化。

## 重构原则

- 没有明确兼容要求时，直接更新数据结构、接口和调用方并删除旧路径，不为历史实现保留转发接口、别名、双写状态或适配层。

## 项目内头文件依赖

- 使用项目内符号时，直接包含其所属头文件，不依赖其他项目头文件的传递包含；修改后通过编译确认依赖完整。
- 该要求只针对 rrs 自身头文件；除非出现实际问题或任务明确要求，不主动整理标准库和第三方库的传递包含。

## 限定符与属性

- 使用 `const` 表达真实的只读约束并保持 const-correctness，不机械修饰所有局部变量、临时值和按值参数。
- 仅当忽略返回值很可能构成错误时使用 `[[nodiscard]]`，不统一添加到普通 getter、比较运算和辅助函数。
- 仅当接口确实需要且完整调用链能够维持不抛异常时使用 `noexcept`；不得机械添加到 getter、简单函数或可能分配内存的函数。
