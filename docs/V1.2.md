# V1.2 Source Layout Refactor

## Goal

V1.2 is a small source layout refactor. It reduces one-class-per-directory fragmentation while preserving clear runtime boundaries.

This iteration does not change server behavior, public header paths, protocol, tick logic, session logic, or threading ownership.

## Completed

- Kept `include/rrs/...` unchanged as the public include boundary.
- Reorganized `src/` by implementation responsibility:
  - `src/app`: process entrypoint
  - `src/runtime`: active runtime components and thread-driven services
  - `src/room`: room simulation and entity state
  - `src/protocol`: inbound/outbound messages and snapshot pipeline data
  - `src/session`: session management
  - `src/support`: config, logging, metrics data, and persistence event types
- Removed empty implementation directories left by earlier iterations.
- Updated CMake source list to match the new layout.

## Current Layout

```text
src/
  app/
    main.cpp
  runtime/
    Acceptor.cpp
    IOThread.cpp
    WorkerThread.cpp
    MetricsReporter.cpp
    PersistenceWorker.cpp
  room/
    Room.cpp
    PlayerEntity.cpp
  protocol/
    InboundMessage.cpp
    OutboundMessage.cpp
    Snapshot.cpp
    SnapshotRingBuffer.cpp
  session/
    SessionManager.cpp
  support/
    AppConfig.cpp
    Logger.cpp
    TickMetrics.cpp
    PersistenceEvent.cpp
```

## Design Notes

- `runtime` groups code that actively drives server execution or owns a service loop.
- `room` remains focused on gameplay-state simulation.
- `protocol` groups data moving between IO, Worker, and snapshot delivery.
- `support` contains infrastructure that should not own main server flow.
- Header paths stay stable to avoid unnecessary include churn.
