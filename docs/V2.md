# V2 Local Message Queue And Ownership

## Goal

V2 verifies the local message path before adding TCP networking.

The important boundary is:

```text
fake IO thread -> WorkerThread inbound queue -> Room pending inputs -> Room tick
```

## Included

- `InboundMessage`
- `PlayerInput`
- `ThreadSafeQueue`
- `WorkerThread::postMessage`
- Worker-side routing from inbound queue to owned rooms
- Room-side pending input storage
- Input consumption only during `Room::tick`
- Fake input thread in `main`

## Not Included

- TCP network
- Real IO thread
- Session
- Reconnect generation
- Outbound queue
- Snapshot generation

## Acceptance

Run the executable and verify:

- Worker still advances rooms at roughly 30Hz
- Fake input messages are accepted through `WorkerThread::postMessage`
- Rooms consume inputs during tick
- Final room state prints consumed input counts

## Ownership Rule

The fake IO thread does not access `Room` directly. It only posts `InboundMessage` to `WorkerThread`.

`Room` state is mutated only by its owner worker loop.
