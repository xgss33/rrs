# V1.4 Getter Inline Refactor

## Goal

V1.4 moves a small set of pure getter functions into headers so the compiler can inline them across translation units without relying on link-time optimization.

This iteration intentionally avoids broad inlining. It only touches functions that directly return member values or cheap container sizes.

## Completed

- Inlined simple `Room` getters:
  - `id`
  - `tickSeq`
  - `nextTickTime`
  - `consumedInputCount`
  - `pendingInputCount`
  - `entityCount`
  - `snapshotHistorySize`
  - `rejectedInputCount`
- Inlined `SnapshotRingBuffer::size` and `SnapshotRingBuffer::capacity`.
- Inlined `IOThread` snapshot counter getters backed by atomic loads.
- Inlined `PersistenceWorker::writtenEventCount`.
- Removed the matching out-of-line definitions from `.cpp` files.

## Not Inlined

- `Room::latestSnapshot` stays out-of-line because it returns a copied `RoomSnapshot`.
- Queue mutation functions stay out-of-line because they change state.
- `SessionManager` query methods stay out-of-line because they traverse or copy containers.
- Message/event factory functions stay out-of-line because they are not currently a measured hot path.
- `toString` helpers stay out-of-line because they are primarily used for logging and diagnostics.

## Design Notes

`inline` does not guarantee that the compiler removes the function call. It makes the function body visible at the call site and permits the compiler to inline it when profitable.

For this project, the useful boundary is pure getter-level code. Larger logic should stay in `.cpp` until profiling proves otherwise.
