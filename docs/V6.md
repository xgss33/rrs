# V6 Broadcast And Outbound Queue

## Goal

V6 adds the reverse message path from worker to IO.

```text
Room tick -> latest snapshot -> WorkerThread OutboundMessage -> IOThread outbound queue -> TCP client
```

## Included

- `OutboundMessage`
- Worker outbound sink callback
- Snapshot text payload generation
- IOThread outbound message queue
- Per-client outbound queue
- Per-client queue limit
- Dropping old snapshot payloads when the queue is full

## Not Included

- Binary snapshot encoding
- Delta snapshot
- Ack
- Resync
- epoll
- Full TCP backpressure handling

## Manual Test

Start the server:

```bash
./build/realtime_room_server
```

In another terminal:

```bash
nc 127.0.0.1 9000
JOIN 1 1001
INPUT 1 0
```

Expected output includes repeated snapshot lines:

```text
SNAPSHOT room=1 tick=<tickSeq> entities=1
```
