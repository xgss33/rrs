# 二进制协议

所有整数使用大端字节序。SessionId 仅在当前服务器进程生命周期内有效。

## Frame

```text
length       uint32
message_type uint8
payload      bytes
```

`length` 包含 message type 和 payload，不包含自身的 4 字节。客户端请求的最大 length 为 9；服务端响应最大为 64 KiB。

## 客户端消息

```text
1 JOIN        player_id:uint64
2 RECONNECT   session_id:uint64
3 INPUT       move_x:int16 + move_y:int16 + input_flags:uint8
4 LEAVE       empty
```

PlayerId 和 SessionId 必须非零。`input_flags` 的 bit 0 表示分裂，其余位必须为 0。

## 服务端消息

```text
101 FULL_SNAPSHOT  session_id:uint64 + snapshot
102 DELTA_SNAPSHOT snapshot
103 ERROR          diagnostic text
```

JOIN 和 RECONNECT 成功后都返回 FULL_SNAPSHOT。普通同步使用 DELTA_SNAPSHOT。最终快照携带 winner：已有同步基线的连接收到 DELTA_SNAPSHOT，恰好在结束 Tick 首次建立基线的连接收到 FULL_SNAPSHOT；发送完毕后服务端关闭连接并清理整局。ERROR 是终止消息，发送后关闭连接；非法 frame 允许不发送 ERROR 直接关闭。

## Snapshot

```text
tick_seq             uint16
flags                uint8
player_update_count  uint8
player updates:
    player_id          uint64
    visible_ball_mask  uint16
    changed_ball_mask  uint16
    changed balls: position_x:int16 + position_y:int16 + radius:uint16
removed_player_count uint8
removed player ids   uint64[]
food_update_count    uint16
food updates: food_index:uint16 + position_x:int16 + position_y:int16
winner_player_id     uint64  // 仅 winner_present 时存在
```

flags 的 bit 0 表示 `winner_present`，其余位必须为 0。FULL_SNAPSHOT 中每个玩家的 changed mask 必须等于 visible mask，removed players 必须为空，并携带完整食物基线；DELTA_SNAPSHOT 在已有基线上应用。
