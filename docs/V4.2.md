# V4.2 IO Session 权威与 WorkerRoomTable 边界收紧

## 目标

修正 IO 本地 session 与 fd 绑定的权威关系，并继续收紧 Worker 房间状态边界。

本版本不处理 outbound reliable/snapshot 策略，该部分留到 tick 和网络同步设计时统一处理。

## IO 调整

- `IoSessionTable::Unbind()` 只有在 `session_id -> fd` 仍然指向当前 fd 时，才会删除本地 session。
- 新增 `IoSessionTable::FindForClient(fd, session_id)`，用于确认当前 fd 仍然拥有该 session。
- `IOThread::SessionForClient()` 改为通过 `FindForClient()` 查询，避免旧 fd 在 reconnect 后继续发送 `INPUT/LEAVE`。
- `IoSessionTable::Find(session_id)` 改为内部辅助接口，外部不能绕过 fd 权威校验直接拿 session。

## Worker 调整

- `WorkerRoomTable` 不再公开 `rooms()` 容器引用。
- `WorkerRoomTable` 不再公开可写的 `FindBinding()`，外部只能读取 binding。
- `WorkerThread` 通过 `WorkerRoomTable::NextWakeTime()` 计算下一次唤醒时间。
- `WorkerThread` 通过 `WorkerRoomTable::TickDueRooms()` 触发 room tick，不直接遍历 room 容器。

## Leave 后处理

- 删除 `UpdateRoomOpenState()` 后再 `DeleteRoomIfEmpty()` 的冗余流程。
- 新增 `WorkerRoomTable::HandleRoomAfterLeave()`：
  - room 为空时删除 room。
  - room 非空时更新 open room 状态。

## 保留项

- outbound queue 满时如何区分 reliable response 和 snapshot 暂不修改。
- `WorkerRoomTable::FindRoom()` 暂时保留，因为 Worker 仍需要调用 Room 的游戏逻辑接口。
- 当前队列仍使用 `Mailbox` 的 mutex 实现。

## 验证

- `cmake --build build` 通过。
