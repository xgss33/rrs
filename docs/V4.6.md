# V4.6 Pressure Entity Scale

## Change

Increase default room food count from `512` to `1024`.

## Current Pressure Profile

- `16` players per room.
- `1024` food entities per room.
- `30Hz` room tick.
- Full snapshot output.

## Reason

`512` food entities made local pressure tests too light. `1024` food entities keeps the gameplay model simple while making collision checks, snapshot encoding, and outbound bandwidth pressure easier to observe during staged load tests.

This is now the default pressure profile for subsequent test records.
