# V2.1.1 - Google C++ Style Cleanup

## 目标

V2.1 已经完成运行时拓扑的第一步调整，本轮不继续推进架构功能，只清理代码风格，避免后续 V2 多线程、epoll、队列拆分时在旧命名风格上继续扩展。

## 本轮完成

- 统一常量、全局变量风格：
  - 常量使用 `kUpperCamelCase`，例如 `kIoThreadCount`。
  - 全局变量使用 `g_lower_snake_case`，例如 `g_stop_requested`。
- 统一类型、函数、变量风格：
  - 类型、类、结构体、枚举类型使用 `UpperCamelCase`。
  - 动作型函数使用 `UpperCamelCase`，例如 `Start()`、`Stop()`、`HandleInput()`。
  - 纯读 accessor 使用 `lower_snake_case`，例如 `tick_stats()`、`session_snapshots()`、`written_event_count()`。
  - 局部变量、函数参数、结构体字段使用 `lower_snake_case`。
  - 类成员变量使用 `lower_snake_case_`。
- 统一枚举值风格：
  - 枚举值使用 `kName`，例如 `InboundMessageType::kPlayerInput`。
- 统一自定义队列接口：
  - `ThreadSafeQueue::push()` 改为 `ThreadSafeQueue::Push()`。
  - STL 容器自身接口保持原样，例如 `push_back()`、`queue_.push()`。
- 统一日志字段命名：
  - 日志中的结构化字段尽量使用 `snake_case`，例如 `over_budget`、`pending_inputs`、`written_events`。

## 不在本轮处理

- 不改变现有 V2.1 的线程拓扑。
- 不引入 epoll、eventfd、全局 session 读写锁。
- 不拆分 IO/Worker 队列矩阵。
- 不补客户端压测或 fake 行为清理。

这些属于后续 V2.2 之后的设计和实现范围。

## 验证

- 只做编译验证。
- 不做端到端客户端测试。
