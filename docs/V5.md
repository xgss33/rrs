# V5 Minimal TCP Text Protocol

## Goal

V5 replaces fake IO input with a real TCP accept/read path.

The important path is:

```text
TCP client -> Acceptor -> IOThread / SessionManager -> WorkerThread -> Room
```

## Protocol

Commands are newline-delimited text:

```text
JOIN <roomId> <playerId>
INPUT <moveX> <moveY>
RECONNECT
QUIT
```
## Included

- `Acceptor`
- TCP listen socket on configured port
- IOThread client fd management
- Minimal line-based command parsing
- Session creation from `JOIN`
- Input routing from TCP client to Worker
- Reconnect generation increment and stale input rejection

## Not Included

- epoll
- Multi IO thread
- Binary packet framing
- Protobuf
- Login/Auth
- Snapshot outbound broadcast
- Slow client backpressure

## Manual Test

Start the server:

```bash
./build/realtime_room_server
```

In another terminal:

```bash
nc 127.0.0.1 9000
JOIN 1 1001
INPUT 1 0
INPUT 1 0
RECONNECT
INPUT 1 0
QUIT
```
