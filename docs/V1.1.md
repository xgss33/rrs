# V1.1 Serverization

## Goal

V1.1 turns the previous V8 demo-oriented executable into a continuously running V1 server process.

This iteration does not add epoll, binary protocol encoding, or production deployment features. Those remain later scalability and engineering extensions.

## Completed

- Server process now runs until `SIGINT` or `SIGTERM` instead of stopping after a fixed demo duration.
- Removed demo/fake configuration fields from server config:
  - `demoRoomCount`
  - `demoRunSeconds`
  - `fakeInputIntervalMs`
  - `fakeReconnectAfterSeconds`
- Kept the V1 TCP text protocol:
  - `JOIN <roomId> <playerId>`
  - `INPUT <moveX> <moveY>`
  - `RECONNECT`
  - `QUIT`
- Server CMake now only builds `realtime_room_server`.
- Bot/load client is split out of the server project and moved to an independent project: `../RealtimeRoomBot`.
- Added mutex protection around worker and IO state snapshots used by metrics reporting, avoiding data races during long-running server execution.

## Current Server Boundary

- `Acceptor` owns passive TCP accept and hands accepted sockets to `IOThread`.
- `IOThread` owns client sockets, sessions, text command parsing, outbound queueing, and worker message submission.
- `WorkerThread` owns rooms, player entities, tick execution, input consumption, and snapshot generation.
- `PersistenceWorker` owns asynchronous event writing.
- `MetricsReporter` only reads snapshots through explicit thread-safe query methods.

Cross-thread state is still accessed through messages or snapshot-copy APIs. Direct shared mutable ownership between IO and Worker remains disallowed.

## Known Limitations

- TCP IO still uses a simple nonblocking polling loop, not epoll.
- Protocol is still line-based text, not Protobuf or binary snapshot encoding.
- Only one worker thread is active even though config keeps `workerThreadCount` for future sharding.
- Room creation is static at startup.
- Reconnect command still injects one stale input intentionally to verify generation rejection.
- Backpressure strategy is still basic bounded outbound snapshot dropping.
- No command-line config or config file loading yet.

## Next Candidate Work

- Add command-line or file-based config loading.
- Replace line protocol with framed protocol.
- Replace polling IO loop with epoll.
- Add room lifecycle state machine.
- Expand independent bot project into a configurable load-test tool.
