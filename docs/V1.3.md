# V1.3 Header Layout Refactor

## Goal

V1.3 reorganizes public header directories to reduce one-module-per-folder fragmentation.

The goal is meaningful interface grouping, not a strict one-to-one match with `src/`.

## Completed

- Kept `include/rrs/core` for low-level shared primitives.
- Added `include/rrs/runtime` for active runtime interfaces:
  - `Acceptor`
  - `IOThread`
  - `WorkerThread`
  - `MetricsReporter`
  - `PersistenceWorker`
- Consolidated protocol-facing data under `include/rrs/protocol`:
  - inbound messages
  - outbound messages
  - snapshots
  - snapshot history
- Consolidated room simulation headers under `include/rrs/room`:
  - room state
  - player entity
  - math types
- Kept `include/rrs/session` for session lifecycle state and manager.
- Added `include/rrs/support` for infrastructure-style headers:
  - app config
  - logger
  - tick metrics data
  - persistence event data
- Removed old fragmented header directories:
  - `config`
  - `game`
  - `io`
  - `log`
  - `message`
  - `metrics`
  - `net`
  - `persistence`
  - `sync`
  - `worker`

## Current Header Layout

```text
include/rrs/
  core/
    ThreadSafeQueue.h
    Types.h
  protocol/
    InboundMessage.h
    OutboundMessage.h
    Snapshot.h
    SnapshotRingBuffer.h
  room/
    MathTypes.h
    PlayerEntity.h
    Room.h
  runtime/
    Acceptor.h
    IOThread.h
    MetricsReporter.h
    PersistenceWorker.h
    WorkerThread.h
  session/
    Session.h
    SessionManager.h
  support/
    AppConfig.h
    Logger.h
    PersistenceEvent.h
    TickMetrics.h
```

## Design Notes

- `runtime` means the interface owns or drives a running service, loop, socket accept path, or cross-thread component.
- `protocol` means the type is transferred between layers or represents sync payload data.
- `room` means the type belongs to simulation/world state.
- `support` means infrastructure dependency, not a domain owner.
- This refactor intentionally keeps `Session` separate because it is a lifecycle domain, not a generic protocol message.

## Behavior Impact

No runtime behavior is intended to change in this iteration. Only header file locations and include paths were changed.
