# V3.4 主干线程消息边界重构

## 目标

将 `tmp` 中确认的协议、Session、队列和 Worker 房间所有权设计落入主干，不保留旧协议兼容。

## 核心调整

- 客户端协议改为 `JOIN`、`RECONNECT`、`INPUT`、`LEAVE`。
- `JOIN_OK` 和 `RECONNECT_OK` 只由 Worker 处理完成后返回，IO 不直接宣布进入游戏成功。
- `Session` 保留 `session_id`、`generation`、`player_id`、`io_thread_id`、`worker_id`。
- `SessionRegistry` 只维护 `session_id -> Session`，不再维护 `player_id -> session_id`。
- 删除全局 `MessageChannels`，改为消费线程拥有自己的 `Mailbox`。
- `IOThread` 持有所有 `WorkerInboxSender`，自己拥有 `IoInbox`。
- `WorkerThread` 持有所有 `IoInboxSender`，自己拥有 `WorkerInbox`。
- 删除旧的 `RoomDirectory` 和 `RoomRouter`，Room 由 Worker 动态管理。

## 线程边界

```text
Acceptor
  -> IOThread.EnqueueAcceptedClient(fd)
  -> eventfd wake IOThread

IOThread
  -> decode client frame
  -> bind fd/session
  -> push IoToWorkerMessage to WorkerInbox

WorkerThread
  -> drain WorkerInbox
  -> create/reconnect/remove PlayerEntity
  -> tick room
  -> push WorkerToIoMessage to IoInbox

IOThread
  -> drain IoInbox
  -> map session to fd
  -> enqueue socket send buffer
```

## Room 动态管理

- 每个 Worker 独立生成 RoomId。
- `JOIN` 时优先选择未满 room。
- 如果没有可加入 room，则创建新 room。
- room 满员后从 open room 列表移除。
- `LEAVE` 后 room 未满则可重新加入。
- 空 room 直接删除。

## 当前取舍

- `WorkerToIoMessage` 不携带 `RoomId`，IO 不理解 room 细节。
- room 信息如果客户端需要，只存在于 Worker 生成的 snapshot payload 中。
- `Mailbox` 当前使用 `mutex + deque`，接口保留 `bool Push()`，后续可替换为 bounded queue 或无锁队列。
- `LEAVE` 不返回确认包，服务端解除 session 并通知 Worker 移除玩家。
