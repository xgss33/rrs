# V4 Fake IOThread And Session Generation

## Goal

V4 introduces the IO thread domain without adding TCP.

The important boundary is:

```text
IOThread owns SessionManager
WorkerThread owns Room and PlayerEntity
IOThread -> WorkerThread only through InboundMessage
```

## Included

- `Session`
- `SessionManager`
- `IOThread`
- `SessionId` and `Generation` in `InboundMessage`
- `BindPlayer` message
- Worker-side player generation view
- Worker rejection of stale player input
- Fake reconnect scenario

## Not Included

- TCP
- Acceptor
- epoll
- Packet framing
- Login/Auth protocol
- Outbound snapshot sending

## Acceptance

Run the executable and verify:

- Worker and IO thread start and stop cleanly
- IO thread creates sessions and binds them to rooms
- Room tick and snapshot generation still work
- Reconnect increments session generation
- Worker rejects at least one stale input from the old generation
- New-generation input continues to update room state
