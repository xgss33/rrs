# V1 Worker And Room Tick

## Goal

V1 verifies the server tick foundation before adding network, session, input, or snapshot logic.

## Included

- `WorkerThread` owning multiple `Room` instances
- `Room` maintaining `nextTickTime` and `TickSeq`
- `std::chrono::steady_clock` based 30Hz tick loop
- Limited catch-up, capped by `maxCatchUpTicks`
- Basic per-room tick metrics

## Not Included

- TCP network
- IO thread
- Session
- Cross-thread message queue
- Player input
- Snapshot generation
- Persistence

## Acceptance

Run the executable and verify:

- The worker starts and stops cleanly
- Each room advances at roughly 30Hz
- A 5 second run produces roughly 150 ticks per room
- Tick stats are printed at shutdown
