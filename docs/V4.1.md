# V4.1 IO/Worker 状态边界重构

## 目标

降低 `IOThread` 和 `WorkerThread` 的主类复杂度，让它们更接近线程主流程，而不是直接维护所有内部状态表。

## 调整内容

- 新增 `IoSessionTable`，集中维护 IO 线程本地的 `session -> fd` 和 `session -> Session` 绑定。
- 新增 `WorkerRoomTable`，集中维护 Worker 内部的 room 列表、open room、session-room 绑定。
- `IOThread` 保留 socket、epoll、协议入口、Worker 消息投递职责。
- `WorkerThread` 保留 Worker 主循环、消息分发、tick 调度、snapshot 投递职责。
- 不修改客户端协议、Session 字段、Mailbox 模型、Room 游戏逻辑。

## 当前边界

```text
IOThread
  -> accept fd queue
  -> epoll read/write
  -> decode client frame
  -> update IoSessionTable
  -> push WorkerInbox
  -> drain IoInbox
  -> send client frame

WorkerThread
  -> drain WorkerInbox
  -> call WorkerRoomTable for room/session binding
  -> run room tick
  -> publish snapshot to IoInbox
```

## 不做的事

- 不拆协议解析层。
- 不拆 socket client send buffer。
- 不引入无锁队列。
- 不新增 fake 测试逻辑。
- 不改变 room 动态创建和空 room 删除策略。

## 验证

- `cmake --build build` 通过。
