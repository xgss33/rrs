# V2.3 - Global SessionRegistry And IO Local Cache

## Goal

V2.3 introduces the global session authority required by multiple IOThreads.

Before this iteration, each IOThread owned its own `SessionManager`. That cannot support reconnect or resume across IOThreads because a session created on one IOThread is invisible to another IOThread.

## Completed

- Added `SessionRegistry`.
- `SessionRegistry` is the authoritative session owner.
- Removed the old local `SessionManager` from the build and source tree.
- Added global maps:

```text
SessionId -> Session
PlayerId -> SessionId
SessionId -> IoThreadId
```

- Added `std::shared_mutex` around global session state:
  - write operations use `std::unique_lock`
  - read operations use `std::shared_lock`

- IOThread now receives a shared `SessionRegistry&`.
- IOThread keeps a local cache:

```text
SessionId -> Session snapshot
```

- Client connection still owns the fd-local binding:

```text
fd -> SessionId
```

- `JOIN` now creates or finds session through `SessionRegistry`.
- At this iteration, the old same-connection `RECONNECT` command still updated generation through `SessionRegistry`.
- Current code has since replaced this runtime command with `RESUME <session_id>` in V2.6.
- Disconnect now marks the global session as `Disconnected`.
- Disconnect also updates the IOThread local cache session state to `Disconnected`.

## Current Session Flow

### JOIN

```text
Client
  -> IOThread
  -> SessionRegistry::CreateOrFindByPlayer(player_id, room_id, io_thread_id)
  -> IOThread local cache stores Session snapshot
  -> fd stores SessionId
  -> Worker receives BindPlayer
```

### INPUT

```text
Client
  -> IOThread
  -> fd finds SessionId
  -> IOThread reads local Session cache
  -> route to owner worker by room_id
```

### Disconnect

```text
socket close
  -> IOThread::CloseClient
  -> SessionRegistry::MarkDisconnected(session_id)
  -> IOThread local cache marks SessionState::kDisconnected
```

### Historical RECONNECT State

At V2.3 time, the old same-connection `RECONNECT` command still existed, but the fake stale-input injection had already been removed.

```text
RECONNECT
  -> SessionRegistry generation update
  -> generation++
  -> SessionState::kConnected
  -> SessionId -> IoThreadId updated
  -> Worker receives BindPlayer with new generation
```

## Intentional Transitional Limits

- Real `RESUME <session_id>` on a new TCP connection is not implemented yet in V2.3. It is implemented by V2.6.
- Acceptor still assigns all accepted fds to the first IOThread.
- IOThread still polls client fds instead of epoll in V2.3. This is replaced by epoll in V2.6.
- Acceptor-to-IO handoff still does not use eventfd.
- Worker output still uses the `io_thread_id` carried by `InboundMessage` in V2.3.
- Worker output does not yet call `SessionRegistry::FindIoThread(session_id)` in V2.3.

That final worker output routing belongs to V2.4.

## Why This Step Matters

V2.2 made the queue topology explicit, but session ownership was still local to IOThread.

V2.3 separates:

- socket ownership: IOThread local
- fd-to-session binding: IOThread local
- session truth: global `SessionRegistry`
- room/player state: WorkerThread local

This is the minimum boundary needed before implementing real cross-IO resume.

## Validation

- Compilation only.
- No end-to-end client test is required for this intermediate V2 step.
